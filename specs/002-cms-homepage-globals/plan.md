# Implementation Plan: CMS-Driven Homepage Globals

**Branch**: `002-cms-homepage-globals` | **Date**: 2026-04-02 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/002-cms-homepage-globals/spec.md`

## Summary

Create 4 Payload CMS globals (HomepageHero, HomepagePillars, HomepageSpecKit, HomepageContact) to make all homepage section content editable from `/admin`. Seed with current hardcoded content during migration. Components fetch their global and fall back to defaults when CMS is unreachable. Roger can update any homepage copy without a code deploy.

## Technical Context

**Language/Version**: TypeScript 6.0.2 + React 19.2.4
**Primary Dependencies**: Payload CMS 3.80 (globals, admin UI, migrations), Next.js 16.2.1 (App Router, server components)
**Storage**: Neon serverless Postgres via `@payloadcms/db-postgres`
**Testing**: Bun test (unit), Playwright 1.58.2 (E2E)
**Target Platform**: Vercel (SSR)
**Project Type**: Marketing website CMS integration
**Performance Goals**: Homepage render time must not regress; CMS fetches should add <100ms
**Constraints**: Must maintain fallback behavior — site works identically if CMS is empty/unreachable. Must generate a Payload migration. No new npm dependencies.
**Scale/Scope**: 4 new global definitions + 4 component updates + 1 config update + 1 migration + 1 seed update
